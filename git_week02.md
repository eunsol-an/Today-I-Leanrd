## Git 2주차 총정리
- 협업을 위한 순서
  - isseu 생성하고 각자 할당 → 각 branch 만들어서 작업 → 기준이 될 branch에 merge
- tracking은 branch 기준
- branch는 특정 commit에서 파생됨
- 같은 파일의 같은 부분을 수정하면 merge conflict가 발생 → 충돌 부분 수정 후 merge commit
